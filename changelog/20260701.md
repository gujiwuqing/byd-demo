## 20:30 包名重命名 + ADB 授权重构（dadb 库替换手写协议）

**修改总结**（47+ 个文件，大规模重构）

### 一、包名重命名（解决安装失败）

**根本原因**：`applicationId` 为 `com.bydlauncher`，与 BYD 系统自带桌面应用冲突，导致 USB/网盘安装时报"解析安装包出现问题"或"不支持安装桌面类应用"。

**修复**：
- `applicationId` 改为 `com.diui.launcher`
- `namespace` 改为 `com.diui.launcher`
- 所有 Java 文件包名从 `com.bydlauncher` 重命名为 `com.diui.launcher`
- 目录结构从 `com/bydlauncher/` 迁移到 `com/diui/launcher/`
- XML layout 中自定义 View 全限定名同步更新（`ArcTempView`、`VehicleDiagramView`）
- `set-home-activity` 命令更新为 `com.diui.launcher/.MainActivity`

### 二、AndroidManifest.xml 修改

1. 移除 `package` 属性（由 build.gradle `namespace` 管理）
2. 移除 `android.intent.category.HOME`（HomeStubActivity 的 intent-filter，避免被识别为桌面类应用）
3. 恢复全部 `BYDAUTO_*` 权限声明（`pm grant` 需要 manifest 声明，byd-apps 项目验证了声明权限不影响安装）
4. 新增 `INTERNET` 权限（dadb 库需要 TCP Socket 连接本地 ADB）

### 三、ADB 授权系统重构（参考 Overdrive 项目）

**改动文件**：

#### build.gradle
- `compileSdk` 34 → 33（避免新资源格式兼容性问题）
- 新增依赖 `dev.mobile:dadb:1.2.8`

#### AdbHelper.java（完全重写）
- 删除手写 ADB 协议实现（CMD_CNXN/CMD_AUTH/CMD_OPEN 等常量、authenticate()、execShell()、sendMessage()、readMessage() 等方法全部移除）
- 改用 dadb 库：`Dadb.create()` 连接、`dadb.shell()` 执行命令
- 新增 `connectAndAuth()` 方法：两步流程，先连接认证（触发车机弹窗），后台轮询等待授权（3秒/次，最多40次=2分钟）
- 新增 `AuthCallback` 接口：`onAuthPending()` / `onAuthGranted()` / `onAuthFailed()`
- 密钥管理改用 dadb 的 `AdbKeyPair.generate()` / `AdbKeyPair.read()`，存储为标准 `adbkey` / `adbkey.pub` 文件

#### AdbKeyManager.java（已删除）
- dadb 库自带完整的 RSA 密钥生成/存储/签名/公钥编码，不再需要手写实现

#### MainActivity.java
- 移除 `import AdbKeyManager`
- `startAdbGrant()` 改为两步流程：`AdbHelper.connectAndAuth()` → 成功后 `AdbHelper.grantPermissions()`
- 授权等待期间显示 Toast 提示用户在弹窗中点击"允许"
- `clearKeys` 调用改为 `AdbHelper.clearKeys()`
