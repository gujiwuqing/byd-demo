# 车机 ADB 连接排查指南

**适用固件**：2406（4.0 UI）/ 2407+  
**目标**：成功执行 `adb connect 192.168.10.10:5555`

---

## 前提条件

`192.168.10.10` 是车机连接到**手机热点**时的固定 IP。只有以下网络拓扑才能用这个 IP：

```
手机（开热点）
    ├── 车机连接手机热点  → 车机 IP = 192.168.10.10
    └── 电脑连接手机热点  → 电脑可以 adb connect 到车机
```

> ⚠️ 如果车机连的是家里 WiFi 或其他热点，IP 不是 192.168.10.10，需要在车机设置里查实际 IP。

---

## 第一步：开启车机 ADB

BYD 固件默认关闭 ADB，需要手动开启（约 30 天后自动关闭，需重新操作）。

### 2406/2407+ 固件开启步骤

```
1. 手机蓝牙连接车机（或直接在车机操作）
2. 打开车机拨号界面
3. 输入：*#91532547#*
4. 记录弹出的 IMEI 号（16 位数字）
5. 手机打开网址：https://ahmada3mar.github.io/BYD/
6. 输入 IMEI → 点击生成 → 得到解锁密码
7. 在车机上输入密码 → 提示"ADB 已开启"即成功
```

**常见失败原因：**
- 拨号界面没反应：确认输入的是 `*#91532547#*`，不是打电话
- 网站生成的密码无效：确认 IMEI 号是否填写完整
- 提示已关闭/无效：固件版本不对，尝试其他密码生成工具

---

## 第二步：确认网络环境

### 推荐方式（手机热点）

```
1. 手机开热点（建议 2.4GHz，车机兼容性更好）
2. 车机连接手机热点
3. 电脑也连接同一个手机热点
4. 确认车机和电脑在同一网段
```

### 查看车机实际 IP（非手机热点场景）

```
车机 → 设置 → 关于本机 → IP 地址
或
车机 → 设置 → 网络 → WiFi → 已连接网络 → 查看详情
```

记录到的 IP 替换下方命令中的地址：
```bash
adb connect <车机IP>:5555
```

---

## 第三步：执行连接

```bash
# 标准连接命令
adb connect 192.168.10.10:5555

# 验证连接成功
adb devices
```

**成功输出：**
```
List of devices attached
192.168.10.10:5555    device
```

**失败输出及处理：**

| 输出信息 | 原因 | 解决方法 |
|---------|------|---------|
| `failed to connect` | 网络不通或 ADB 未开 | 检查网络拓扑和第一步 |
| `connection refused` | ADB 服务未启动 | 重新执行开启 ADB 步骤 |
| `offline` | 连接过期或需要授权 | 在车机上点"允许调试" |
| 无响应/超时 | IP 错误或防火墙 | 查看车机实际 IP |

---

## 第四步：授权 ADB

首次连接时，**车机屏幕会弹出"是否允许 USB 调试"弹窗**：

```
1. 在车机屏幕上点击"允许"
2. 建议勾选"始终允许来自此计算机"
3. 回到电脑执行 adb devices 确认状态为 device
```

> 如果错过弹窗，执行 `adb disconnect && adb connect 192.168.10.10:5555` 重新触发。

---

## 第五步：用迪加 APP 辅助确认（可选）

如果已安装迪加 APP，可以通过它确认 ADB 状态：

- 迪加参数 ID **1101** = 无线 ADB 开关状态
- 在迪加的参数查看界面输入 1101 查看当前值
  - `1` = ADB 已开启
  - `0` = ADB 已关闭

---

## 常用 ADB 命令备查

```bash
# 连接车机
adb connect 192.168.10.10:5555

# 查看已连接设备
adb devices

# 安装 APK 到车机
adb -s 192.168.10.10:5555 install -r app-debug.apk

# 查看实时日志
adb -s 192.168.10.10:5555 logcat -s BydVehicleManager:I BydEnvDetector:I

# 进入车机 shell
adb -s 192.168.10.10:5555 shell

# 断开连接
adb disconnect 192.168.10.10:5555

# 查看所有已安装包（找高德包名）
adb -s 192.168.10.10:5555 shell pm list packages | grep -iE "autonavi|gaode|amap|navi"

# 手动授权 BYD 权限
adb -s 192.168.10.10:5555 shell pm grant com.bydlauncher android.permission.BYDAUTO_AC_GET
```
