# 高德地图悬浮版调试指南

**目标**：找到高德悬浮版的正确启动方式，让迪UI无界模式右侧真正显示悬浮地图。

**前提**：手机/电脑和车机连接同一个 WiFi 热点，ADB 已连接。

---

## 第一步：连接车机 ADB

```bash
adb connect 192.168.10.10:5555
adb devices
# 预期看到：192.168.10.10:5555  device
```

---

## 第二步：找到高德悬浮版的包名

```bash
# 列出所有包含 "auto" "gaode" "amap" "navi" 关键词的包
adb shell pm list packages | grep -iE "autonavi|gaode|amap|navi|map"
```

**把输出结果记录下来**，找到类似以下格式的包名：
- `package:com.autonavi.minimap`（标准版）
- `package:com.autonavi.amapauto`（车机版）
- `package:com.autonavi.amapautolite`（精简版）
- 或者其他自定义包名（魔改/定制版）

---

## 第三步：查看该应用的所有组件

把上一步找到的包名替换到下面命令中（以 `YOUR_PKG` 代替）：

```bash
# 3a. 查看所有 Activity
adb shell dumpsys package YOUR_PKG | grep "Activity{" | head -30

# 3b. 查看所有 Service
adb shell dumpsys package YOUR_PKG | grep "Service{" | head -30

# 3c. 查看所有已导出组件（exported=true 才能从外部调用）
adb shell dumpsys package YOUR_PKG | grep "exported=true"

# 3d. 查看 Intent Filter（找 float/window/overlay 相关 Action）
adb shell dumpsys package YOUR_PKG | grep -A5 "IntentFilter"
```

**重点关注**名字中包含以下关键词的组件：
- `Float` / `Floating`
- `Window` / `WindowMode`
- `Pip` / `PictureInPicture`
- `Overlay` / `Mini`
- `Car` / `Auto`

---

## 第四步：尝试不同的启动方式

把找到的 Activity/Service 全名替换到命令中（格式：`包名/类名全路径`）

### 尝试方式 A：直接启动某个 Activity

```bash
# 尝试启动名字包含 Float 的 Activity
adb shell am start -n YOUR_PKG/FloatActivity全路径

# 例如
adb shell am start -n com.autonavi.minimap/com.autonavi.minimap.FloatMainActivity
```

### 尝试方式 B：启动 Service（悬浮窗通常由 Service 控制）

```bash
adb shell am startservice -n YOUR_PKG/FloatService全路径
```

### 尝试方式 C：带参数启动（触发悬浮模式）

```bash
# 常见的悬浮模式触发参数
adb shell am start -n YOUR_PKG/主Activity \
  --ez "float_mode" true

adb shell am start -n YOUR_PKG/主Activity \
  --ei "launch_mode" 1

adb shell am start -n YOUR_PKG/主Activity \
  --es "window_mode" "float"
```

### 尝试方式 D：发送广播

```bash
# 如果发现有 Receiver 组件
adb shell am broadcast -a "com.autonavi.FLOAT_MODE" -n YOUR_PKG/ReceiverFullName
```

---

## 第五步：观察窗口状态

每次尝试启动后，执行以下命令查看当前窗口信息：

```bash
adb shell dumpsys window windows | grep -E "Window\{|packageName|type=" | head -30
```

**判断标准**：
- 如果看到 `type=TYPE_APPLICATION_OVERLAY` 或 `type=2038` → 成功创建悬浮窗 ✅
- 如果看到 `type=TYPE_BASE_APPLICATION` 或 `type=1` → 普通全屏启动 ❌

## 附：快速诊断脚本（一键执行）

把以下内容保存为 `check_amap.sh`，替换 `YOUR_PKG` 后执行：

```bash
#!/bin/bash
PKG="YOUR_PKG"

echo "===== 包名：$PKG ====="
echo ""
echo "===== Activity 列表 ====="
adb shell dumpsys package $PKG | grep "Activity{" | head -20

echo ""
echo "===== Service 列表 ====="
adb shell dumpsys package $PKG | grep "Service{" | head -20

echo ""
echo "===== 已导出组件 ====="
adb shell dumpsys package $PKG | grep "exported=true" | head -20

echo ""
echo "===== 当前窗口状态 ====="
adb shell dumpsys window windows | grep -E "Window\{|packageName|type=" | grep $PKG | head -10
```

执行方式：
```bash
chmod +x check_amap.sh
./check_amap.sh
```
