# BYD Launcher UI 重构设计文档

**日期**: 2026-06-28
**状态**: 已确认

## 1. 项目背景

BYDLauncher 是一个 Android Launcher 应用，目标是安装到 BYD 汽车 12-13 英寸横屏车机上替换原生 Launcher。

### 当前问题

- 三栏横向布局（车辆状态 + 空调控制 + 应用列表）信息密度过高，视觉拥挤
- 应用列表常驻占据 3/8 屏幕，使用频率低但空间占用大
- 空调控制面板按钮密集（风量 8 个按钮一排）
- 所有功能平铺，缺乏信息层级

### 设计目标

- 新势力简洁风格：大卡片、大字体、充分留白
- 简洁桌面 + 卡片弹出：主屏只展示关键信息，其他功能按需弹出
- 适配 12-13 英寸横屏车机

## 2. 整体架构

采用"双区桌面 + 底部 Dock"布局，分为三个层级：

```
┌─────────────────────────────────────────────────────────────────┐
│                     主内容区（85% 高度）                         │
│   ┌──────────────────┐    ┌──────────────────────────────┐     │
│   │   左区（40%）     │    │      右区（60%）              │     │
│   │   时钟 + 留白     │    │      车辆状态大卡片           │     │
│   └──────────────────┘    └──────────────────────────────┘     │
├─────────────────────────────────────────────────────────────────┤
│                     Dock 栏（15% 高度，64dp）                   │
│   [🔒锁车] [❄️空调] [🔄循环] [📦后备箱]  │  [⬆ 所有应用]      │
└─────────────────────────────────────────────────────────────────┘

叠加层（按需弹出，覆盖主内容区，不覆盖 Dock）：
  - 空调控制面板（半屏，从底部滑出）
  - 应用抽屉（全屏，从底部滑出）
```

## 3. 主屏布局

### 3.1 左区（40% 宽度）

- 大号时钟：时间 60sp sans-serif-light，日期 18sp 普通字重
- 左上角对齐，上方留适当边距
- 下方空间保持留白（后续可扩展放车辆图片、天气组件等）

### 3.2 右区（60% 宽度）

一整块车辆状态大卡片，圆角 16dp 深色背景，内部 2 列网格布局：

| 行   | 左列             | 右列             |
|------|-----------------|-----------------|
| 第1行 | ⚡ 电量 78%      | 🛣 续航 326km   |
| 进度条 | 电量进度条（横贯卡片宽度）              |
| 第2行 | 📏 里程 15832km  | ⚡ 电源 READY   |
| 第3行 | 🌡 室外 26°C     | 🚪 全部关闭      |
| 第4行 | 🔒 已锁车        |                  |

- 数值字体 28sp bold，标签字体 14sp 次级灰色
- 电量进度条横贯卡片宽度，根据电量值变色（高=绿、中=橙、低=红）

## 4. 底部 Dock 栏

固定在屏幕底部，高度 64dp，深色半透明背景。水平分为左右两部分：

### 4.1 快捷操作区（左侧 70%）

4 个图标按钮，等间距排列，每个包含图标（24sp emoji）+ 文字标签（12sp）：

| 按钮   | 图标 | 行为                                 |
|--------|------|--------------------------------------|
| 锁车   | 🔒   | 即时操作：点击切换锁定/解锁，状态实时更新 |
| 空调   | ❄️   | 展开操作：点击弹出空调控制面板          |
| 循环   | 🔄   | 即时操作：点击直接切换内/外循环         |
| 后备箱 | 📦   | 预留功能位                            |

- 按钮有按压态反馈（背景微亮）
- 状态激活时图标/文字变为强调色（#00D4FF）

### 4.2 应用入口（右侧 30%）

- 上箭头图标 + "所有应用" 文字
- 点击弹出全屏应用抽屉
- 左侧用竖线与快捷操作区分隔

## 5. 空调控制面板（弹出式）

点击 Dock "空调" 按钮后，从 Dock 栏上方滑出，覆盖主内容区约 60% 高度。

### 5.1 面板结构（从上到下）

1. **拖拽条**：顶部居中，宽 40dp 圆角短条，提示可拖拽关闭
2. **标题行**：左侧 "空调控制" 标题（18sp bold），右侧 ON/OFF 开关按钮
3. **温度区**：居中大号温度显示（40sp bold 强调色），左右 ± 圆形按钮（48dp），下方小字显示当前模式摘要（如 "自动 · 内循环"）
4. **风量条**：标签 "风量" + 8 个等宽按钮（0-7）横排，选中项强调色高亮
5. **底部控制行**：水平排列三组按钮
   - 出风模式：吹面 / 吹脚 / 除雾（3 个按钮）
   - 循环模式：内循环 / 外循环（2 个按钮）
   - 控制模式：自动 / 手动（2 个按钮）

### 5.2 交互

- 所有操作即时生效，无需确认
- 关闭方式：点击遮罩区域 / 再次点击 Dock "空调" / 向下滑动面板
- Dock 栏在面板打开时仍然可操作

## 6. 应用抽屉（全屏弹出）

点击 Dock "所有应用" 按钮后，全屏滑出覆盖主内容区。

### 6.1 布局

- 顶部拖拽条 + "所有应用" 标题（18sp bold）
- 图标网格：6 列，RecyclerView
- 图标尺寸 56dp，下方应用名称（12sp），垂直可滚动
- 排除自身（BYDLauncher）不显示

### 6.2 交互

- 点击图标启动对应应用
- 关闭方式：向下滑动 / 再次点击 Dock "所有应用"
- 与空调面板互斥：打开应用抽屉自动关闭空调面板，反之亦然

## 8. 技术实现要点

### 8.1 架构

- 保持单 Activity 架构（`MainActivity`）
- 不引入 Fragment，使用 View 显隐 + 动画控制弹出面板
- 保持现有 API 层（`BydAcApi`、`BydBodyworkApi`、`BydStatisticApi`）和模拟模式不变
- 只重构 UI 层（布局文件 + MainActivity 的 UI 逻辑）

### 8.2 布局文件规划

| 文件                      | 说明                      |
|--------------------------|---------------------------|
| `activity_main.xml`       | 主 FrameLayout 容器，叠加层 |
| `view_home.xml`           | 主屏内容（时钟 + 车辆卡片） |
| `view_dock.xml`           | 底部 Dock 栏              |
| `view_ac_panel.xml`       | 空调控制弹出面板           |
| `view_app_drawer.xml`     | 应用抽屉                  |
| `item_app.xml`            | 应用图标网格项（复用改造）  |

### 8.3 主 Activity 布局结构

```xml
<FrameLayout>  <!-- 根容器 -->
    <LinearLayout orientation="vertical">  <!-- 主屏 -->
        <include layout="view_home" weight="1" />
        <include layout="view_dock" height="64dp" />
    </LinearLayout>

    <View id="overlay_mask" />  <!-- 半透明遮罩 -->
    <include layout="view_ac_panel" />  <!-- 空调面板，初始隐藏 -->
    <include layout="view_app_drawer" />  <!-- 应用抽屉，初始隐藏 -->
</FrameLayout>
```

### 8.4 需要删除的旧文件

- `view_vehicle_status.xml` → 合并到 `view_home.xml`
- `view_ac_control.xml` → 替换为 `view_ac_panel.xml`
- `view_quick_switch.xml` → 功能移入 Dock 栏

### 8.5 保持不变的文件

- `BydAcApi.java`、`BydBodyworkApi.java`、`BydStatisticApi.java`
- `BydDoorLockApi.java`、`BydPermissionContext.java`、`ReflectionHelper.java`
- `BydVehicleManager.java`
- `VehicleStatus.java`
- `AppListAdapter.java`（改为 6 列网格，图标尺寸调大）
