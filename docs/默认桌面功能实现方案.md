# 默认桌面功能实现方案

**创建日期**: 2026-07-01
**状态**: 待实施

---

## 目标

让迪UI 能够设置为 BYD 车机的默认桌面，按 HOME 键回到迪UI 而非原车桌面。

## 约束

- BYD DiLink 系统在安装 APK 时会扫描 Manifest，检测到 `CATEGORY_HOME` 直接拒绝安装
- 不确定 `android:enabled="false"` 的 Activity 能否绕过此检测

---

## 实施步骤

### 第一步：实现 ADB 备选方案（零风险，不改 Manifest）

利用现有 `AdbHelper.execShell()` 执行 `pm set-home-activity` 命令，在设置页新增"设为默认桌面"按钮。

**涉及文件**：

| 文件 | 改动 |
|------|------|
| `AdbHelper.java` | 新增 `setAsDefaultLauncher(Context, Runnable onSuccess, Runnable onFail)` 方法 |
| `SettingsPage.java` | 新增"设为默认桌面"按钮及点击事件，调用 AdbHelper |
| `page_settings.xml` | 新增按钮布局 |
| `strings.xml` | 新增相关文案 |

**实现要点**：

1. `AdbHelper` 中新增方法，通过 ADB 连接执行：
   ```bash
   # 先尝试 cmd package（Android 9+ 推荐）
   cmd package set-home-activity com.diui.launcher/.MainActivity
   
   # 如果失败，降级尝试 pm
   pm set-home-activity com.diui.launcher/.MainActivity
   ```
2. 执行前检查 `isAdbAvailable()`，不可用时提示用户
3. 执行后验证是否成功（可通过 `cmd package resolve-activity --brief -c android.intent.category.HOME` 检查当前默认桌面）
4. 成功后 Toast 提示，失败后给出排查建议

**待验证**：
- [ ] `pm set-home-activity` 在 Android 9（DiLink 4.0）上是否存在
- [ ] `cmd package set-home-activity` 在 Android 9 上是否可用
- [ ] 如果两者都不可用，尝试其他命令（如 `pm set-installer` 或直接写 `preferred-activities.xml`）

---

### 第二步：验证 Stub Activity 方案能否绕过安装检测

打一个测试包推到真车验证。不写任何逻辑，只验证能否安装。

**改动**（仅测试用，验证后回滚或保留）：

在 `AndroidManifest.xml` 中添加：

```xml
<activity
    android:name=".HomeStubActivity"
    android:enabled="false"
    android:exported="true"
    android:theme="@android:style/Theme.NoDisplay">
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.HOME" />
        <category android:name="android.intent.category.DEFAULT" />
    </intent-filter>
</activity>
```

创建一个空的 `HomeStubActivity.java`：

```java
public class HomeStubActivity extends AppCompatActivity {
    @Override
    protected void onResume() {
        super.onResume();
        startActivity(new Intent(this, MainActivity.class));
        finish();
    }
}
```

**验证方式**：
1. 打包 APK
2. 推到 BYD 真车安装
3. 观察结果：
   - **能安装** → 方案可行，继续第三步
   - **不能安装（提示无法安装桌面类应用）** → 方案不可行，放弃，只保留 ADB 方案

---

### 第三步：完善 Stub Activity 方案（仅在第二步验证通过后执行）

在设置页的"设为默认桌面"按钮中，优先使用 stub Activity 方案，ADB 作为降级。

**涉及文件**：

| 文件 | 改动 |
|------|------|
| `AndroidManifest.xml` | 保留第二步添加的 `HomeStubActivity`（`enabled="false"`） |
| `HomeStubActivity.java` | 保留第二步创建的跳转逻辑 |
| `SettingsPage.java` | "设为默认桌面"按钮逻辑改为先尝试 stub 方案，失败再走 ADB |
| `MainActivity.java` | 新增 `setOnDefaultLauncherListener` 回调 |

**实现要点**：

1. 点击"设为默认桌面"按钮时：
   ```java
   // 动态启用 HomeStubActivity
   getPackageManager().setComponentEnabledSetting(
       new ComponentName(this, HomeStubActivity.class),
       PackageManager.COMPONENT_ENABLED_STATE_ENABLED,
       PackageManager.DONT_KILL_APP
   );
   
   // 触发系统桌面选择器
   Intent homeIntent = new Intent(Intent.ACTION_MAIN);
   homeIntent.addCategory(Intent.CATEGORY_HOME);
   startActivity(homeIntent);
   ```
2. 用户选择迪UI 并点"始终"后，即设为默认桌面
3. 设置页提供"恢复原车桌面"按钮，动态禁用 stub Activity 并触发 HOME intent

---

## 开机自启兜底（可选，独立于上述方案）

无论使用哪种默认桌面方案，都建议实现开机自启作为兜底：

```xml
<receiver android:name=".BootReceiver" android:exported="true">
    <intent-filter>
        <action android:name="android.intent.action.BOOT_COMPLETED" />
    </intent-filter>
</receiver>
```

开机后延迟启动 MainActivity，确保即使默认桌面设置被系统重置（BYD 约 30 天清除权限），迪UI 仍能自动显示。

---

## 风险与回退

| 风险 | 应对 |
|------|------|
| `pm set-home-activity` 在 Android 9 上不存在 | 尝试其他命令，或仅作引导用户手动操作 |
| stub Activity 导致无法安装 | 回滚 Manifest 改动，仅保留 ADB 方案 |
| BYD 30 天清除默认桌面设置 | 开机自启兜底 + 定期检测并重新设置 |
| 用户误设默认桌面后无法恢复 | 设置页提供"恢复原车桌面"功能 |
