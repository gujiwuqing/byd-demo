# 悬浮窗地图集成调研

## 目标

找到魔改高德地图的包名、组件信息，确定如何触发悬浮窗模式，以便 Launcher 启动后以分栏形式展示（左侧车辆状态 + 右侧悬浮地图）。

## 第一步：查找包名

连接车机 ADB 后执行以下命令，找到高德的包名：

```bash
# 按关键词搜索
adb shell pm list packages | grep -i auto
adb shell pm list packages | grep -i gaode
adb shell pm list packages | grep -i map
adb shell pm list packages | grep -i navi
```
com.autonavi.amapauto
启动活动是com.autonavi.auto.remote.fill.UsbFillActivity

找到包名后，后续命令中的 `PACKAGE_NAME` 替换为实际包名。

## 第二步：查看 Activity 列表

```bash
# 列出所有 Activity
adb shell dumpsys package PACKAGE_NAME | grep "Activity{" 

# 查看完整 Activity 声明（含 intent-filter）
adb shell dumpsys package PACKAGE_NAME | grep -B2 -A10 "Activity{"
```

重点关注：
- 有没有名字带 `Float`、`Window`、`Pip`、`Overlay`、`Mini` 的 Activity
- 有没有 `resizeableActivity` 属性
- 有没有 `launchMode` 特殊设置

## 第三步：查看 Service 列表

```bash
# 列出所有 Service
adb shell dumpsys package PACKAGE_NAME | grep "Service{"

# 查看完整 Service 声明
adb shell dumpsys package PACKAGE_NAME | grep -B2 -A10 "Service{"
```

重点关注：
- 有没有名字带 `Float`、`Window`、`Overlay` 的 Service（悬浮窗可能通过 Service 控制）

## 第四步：查看 BroadcastReceiver

```bash
adb shell dumpsys package PACKAGE_NAME | grep "Receiver{"
adb shell dumpsys package PACKAGE_NAME | grep -B2 -A10 "Receiver{"
```

重点关注：
- 有没有名字带 `Float`、`Mode`、`Pip` 的 Receiver（可能通过广播切换模式）

## 第五步：查看权限声明

```bash
adb shell dumpsys package PACKAGE_NAME | grep -A2 "uses-permission"
```

看它声明了哪些权限，尤其是 `SYSTEM_ALERT_WINDOW` 相关的。

## 第六步：查看已导出的组件

```bash
# 查看 exported 的 Activity
adb shell dumpsys package PACKAGE_NAME | grep -E "Activity.*exported=true"

# 查看 exported 的 Service  
adb shell dumpsys package PACKAGE_NAME | grep -E "Service.*exported=true"

# 查看 exported 的 Receiver
adb shell dumpsys package PACKAGE_NAME | grep -E "Receiver.*exported=true"
```

`exported=true` 的组件才能从外部调用。

## 第七步：尝试启动方式

### 7a. 普通启动（看是否默认就是悬浮窗）

```bash
adb shell am start -n PACKAGE_NAME/主Activity全路径
```

### 7b. 带 Extra 参数启动

```bash
# 常见的悬浮窗触发参数
adb shell am start -n PACKAGE_NAME/主Activity --ez "float_mode" true
adb shell am start -n PACKAGE_NAME/主Activity --ez "pip_mode" true
adb shell am start -n PACKAGE_NAME/主Activity --ez "window_mode" "float"
adb shell am start -n PACKAGE_NAME/主Activity --ei "launch_mode" 1
```

### 7c. 指定启动区域（API 24+ 分屏）

```bash
adb shell am start -n PACKAGE_NAME/主Activity --activity-flags 0x04000000
```

`0x04000000` = `FLAG_ACTIVITY_LAUNCH_ADJACENT`

### 7d. 启动特定 Activity（如果发现 Float/Window 类名的 Activity）

```bash
adb shell am start -n PACKAGE_NAME/发现的FloatActivity全路径
```

### 7e. 发送广播触发悬浮窗

```bash
# 如果发现相关 Receiver
adb shell am broadcast -a BROADCAST_ACTION -n PACKAGE_NAME/Receiver全路径
```

### 7f. 通过 Intent Action 启动

```bash
# 查看高德支持的所有 Intent Action
adb shell dumpsys package PACKAGE_NAME | grep -A3 "IntentFilter"
```

## 第八步：查看运行中的窗口信息

启动高德后，查看当前窗口状态：

```bash
adb shell dumpsys window windows | grep -E "Window\{|packageName" | head -20
```

可以看到窗口的 bounds（位置/大小）、类型（type）、是否全屏等信息。

## 关键线索

如果找到以下关键词，基本可以确定悬浮窗触发方式：

| 关键词 | 含义 |
|--------|------|
| `Float` / `Floating` | 悬浮窗 Activity 或 Service |
| `Pip` / `PictureInPicture` | 画中画模式 |
| `Mini` / `MiniMap` | 小窗地图 |
| `Overlay` | 覆盖层 |
| `Window` / `WindowMode` | 窗口模式 |
| `resizeable` | 可调整大小 |
| `freeform` | 自由窗口 |
